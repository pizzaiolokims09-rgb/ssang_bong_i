1. System Overview (시스템 개요)
Bot Name: Ssangbong Bot V2 (AI 기반 하이브리드 자동매매 봇)
Core Goal: 데이트레이딩(5~7% 복리)을 베이스로 하되, 기술적 모멘텀과 심층 펀더멘털 분석을 결합하여 수익을 극대화한다.
Advanced Logic:
1차: 통합 검색망(Base Screener)으로 주도주 포착.
2차: AI Router(Flash)가 차트/수급 기반 트랙(Track A~F) 결정.
3차: AI Fundamental(Pro)이 DART/뉴스 기반 심층 재무 & 리스크 스캔 진행 (오버나잇/스윙의 경우).
Infrastructure: Python 3.11+, Linux Server, KIS API, Telegram.

2. Architecture & Modules (모듈 아키텍처)
kis_client.py: KIS API 통신, 토큰 갱신(403 재시도/백오프 + 스레드 직렬화) 및 잔고 조회 (체결강도 비율 계산, 상장일, 네이버 재무 크롤링, 일봉 차트 3분 TTL 캐시 지원).
telegram_bot.py: 매매 알림 및 2단계 컨펌(/비상정지 등).
signals.py: Phase 1 통합 베이스 조건검색, 트랙별 영웅문 정량 필터 및 보조 지표 연산(VWAP, ATR).
quant_indicators.py: Wilder's Smoothing 정통 퀀트 지표 엔진 (RSI/ATR/ADX/MACD). TradingView 동일 계산 방식 적용. 4대 매도 트리거 통합 판정 함수 제공.
ai_router.py: Phase 2 분봉/호가 분석 및 Track A~F 동적 라우팅 + ATR 기반 동적 TP/SL 산출 (Gemini 3.1 Pro Preview 메인, Gemini 3.5 Flash 폴백).
ai_fundamental.py: Phase 3 공시 및 재무 심층 검증 + Fail-Close 최종 관문 (Gemini 3.1 Pro Preview 메인, Gemini 3.5 Flash 폴백).
orders.py: 포지션 사이징, 5대 절대 방어장치 및 일일 손절/재진입 쿨다운 제어. ATR TP/SL 데이터 포지션 내 저장. 수수료/거래세/슬리피지(ask1/bid1) 반영 손익 계산, 일일 상태(손익/손절카운터/킬스위치) 파일 영속화, 스레드 락(RLock) 기반 주문 직렬화.
monitor.py: 수익률 감시, 4대 매도 트리거(패닉셀/MACD(일봉)/데드크로스/RSI), ATR 2차 안전장치, 분할익절 후 잔여물량 트레일링 스탑, 거미줄 피라미딩 감시. (※ 12:00 올킬/점심정리 로직은 사용자 요청으로 제거됨)
journal.py: SQLite3 매매일지 저장 및 AI 기반 복기. SKIP 후보 섀도 기록(반사실 ML 데이터) 및 장 마감 후 종가 평가.
run.py: 논블로킹 1초 이벤트 메인 루프 (시간대별 진입 필터링, 상한가 차단, Phase 3 gate_check 통합) + 포지션 감시 전용 스레드(10초 주기, AI 추론과 독립) + AI SKIP 판정 15분 캐시.

4. Signal Logic: 3-Phase Routing & Verification
[Phase 1: Base Screener & Track Pre-Filter]
제외 종목: 관리종목, 우선주, ETF, 스펙(SPAC) 원천 제외.
듀얼 유니버스: (1) 거래량/거래대금 상위 150 (돌파형 A/C/G용), (2) pykrx 기관/외인 순매수 상위 수급주 (눌림목 B / 장기 F용, 5분 캐시). 눌림목 후보는 '오늘 조용한 종목'이라 거래량 상위에 잡히지 않으므로 독립 유니버스로 보강.
볼륨 확증 필터(1.5배)는 돌파형 트랙(A/C)에만 적용. 눌림목(B)/매집(D)/장기(F)는 거래량 감소가 전제이므로 면제.
글로벌 테마 관련주는 Phase 2 분석 우선순위 부여 + Phase 3 세그먼트 팩트체크 연동.
[Phase 2: AI Dynamic Routing (Track A~G)]
Track A: 데이트레이딩 (엔벨로프 20, 12.5% 발산 돌파) → God Mode 폐지. 피벗 스나이퍼 경로도 엔벨로프 발산 + AI의 Track A 판정 + 당일 VWAP 이상 + 저변동성 필터(1분 ATR ≥ 가격의 0.15%) + ML 게이트 전부 통과 필수.
Track B: 눌림목 단기 스윙 (거래량 급감(하루치 환산 비교), 5/10일선 근접, 정배열). 물타기 없이 단일 진입(슬롯 예산 전액), 손절은 MA20/SMC 스윙로우.
Track C: 종가 베팅 (14:45~ 수급 폭발, 체결강도 100 이상(매수 체결 우위), 최근 거래량 급증 이력 + 당일 거래량 안정화).
Track D: 세력주 매집 (20봉 신저가, PER 1+ , 유보율 200%+, 자산증감 10%+).
Track E: 폭락주 스나이핑 (250거래일 이상 상장주, 바닥 대비 300%+ 대시세 이력 필수).
Track F: 메가 트렌드 장기 눌림목 (60일 내 50%+ 시세분출 + 150/200일선 근접 + 시대중심 섹터 우량주). God Mode 금지, 종가 분할매집.
Track G: CCI & MACD 더블 모멘텀 스윙 (일봉 기준 CCI(50) 0선 돌파(마진 +10 요구) 및 MACD 동시 0선 상향 돌파, 거래대금 500억 이상). 추세 반전 초입 타점. 14:30 이후 판정(리페인트 방지).
[Phase 3: AI Fundamental Deep Scan + Fail-Close]
Track B, C, D, E, F '오버나잇' 전 필수 통과.
상폐 리스크 스캔 (Track E 전용): 횡령, 배임, 자본잠식 등 6대 악재 실시간 체크.
메가 트렌드 시대 중심주 팩트체크 (Track F 전용): 단순 테마주가 아닌 시대 중심 섹터(반도체/2차전지/로봇/전력/태양광) 우량주인지 AI 검증. 부적합 시 REJECT.
Fail-Close 최종 관문 (전 트랙): 월스트리트 20년 수석 트레이더 페르소나가 거시경제/뉴스 감성/데이터 모순/변동성 리스크를 최종 검토. 확신 70% 미만 시 무조건 REJECT.

5. Security & Risk Management (5대 절대 방어 + 퀀트 방어 체계)
Volume Confirm: 최근 20봉 평균 대비 1.5배 이상 거래량 터치 필수.
Track E Hard Lock: 한 종목 및 전체 Track E 비중이 원금의 10%를 절대 초과하지 않도록 락.
Spider Web Buying (Track E): 200일 최고가 기준 48% / 39% / 34% / 30% 지지선 도달 시 3.75%씩 기계적 분할 매수.
Absolute Hard SL: 단타는 ATR 기반 동적 손절(pivot_low - ATR * 1.5, 최소 -2% ~ 최대 -5% 제한), 스윙 -7%, 폭락주 평단 대비 +5% 도달 시 기계적 전량 익절.
ATR 2차 안전장치: 진입 시 '일봉' Wilder ATR(14)을 기준으로 atr_sl_price(진입가 - 2*ATR), atr_tp_price(진입가 + 3*ATR) 자동 산출. 기존 동적 SL(5분봉 눌림목 저점)을 1차로 사용하고, ATR SL을 최대 허용 손실 한도(2차 안전장치)로 운용하는 이중 방어 구조. 2차 한도는 항상 1차 동적 SL보다 아래에 위치하도록 클램핑 (1분봉 ATR 사용 시 -0.3%에 박혀 1차보다 타이트해지던 결함 수정, 2026-06-01 RISE 사례). 동적 SL이 현재가 위에 놓이는 경우(Track D 등)는 현재가 -5%로 강제 보정.
4대 매도 트리거 (분할익절 후 잔여 포지션 청산): (1) 패닉셀 - 거래량 3배 폭발 음봉 → 무조건 즉시 청산, (2) MACD 음전환(일봉 히스토그램 기준, 분봉은 KIS API 30봉 한계로 계산 불가) → 추세 전환 시그널, (3) 데드크로스 → ADX 약세(< 20)일 때만 실행, (4) RSI 과매수 → ADX 약세(< 20)일 때만 실행.
잔여물량 트레일링 스탑: 50% 분할익절 후 잔여 물량은 고정 % 익절을 적용하지 않고, 최고 수익률 대비 트레일 폭 이탈 시 청산하여 추세를 끝까지 추종한다 (하방은 본절선 +0.5%가 방어). 트레일 폭은 일봉 ATR 비례(1.5×ATR%, 2~6% 클램프)로 변동성에 적응.
상한가 대응: 상한가(+29.5%) 도달 시 매수/매도 잔량비 5배 이상 '잠김'이면 보유 유지(익일 갭 기대, Track A도 EOD 예외 오버나잇 홀드), 풀림/약한 잠김이면 즉시 매도.
수수료/슬리피지 모델: 시장가 매수는 매도1호가(ask1), 시장가 매도는 매수1호가(bid1) 체결로 간주하고, 위탁수수료(편도 0.015%) + 증권거래세(매도 0.15%)를 실현손익에서 차감하여 모의투자 통계의 현실성을 확보한다.
일일 상태 영속화: 당일 실현손익, Track A 손절 카운터, 킬스위치 상태를 data/daily_state.json에 날짜와 함께 저장하여, 봇 재시작에도 일일 손실 한도와 손절 4회 차단이 유지된다.
Track F Weight Lock: 한 종목 및 전체 Track F 비중이 원금의 20%를 절대 초과하지 않도록 락. 200일선 하향 이탈 시 기계적 손절.
Track G Weight Lock & Hold-to-TP: 전체 원금의 10% 비중 락. 일반적인 % 손절/익절 로직을 전면 배제하고, 오직 진입일 일봉 저가(Hard SL), ATR 동적 손절, MACD 데드크로스로만 청산(추세 추종).
Kill Switch: 일일 손실 한도 도달 시 전체 가동 중단.
Risk Hard Guard: 상한가(+25% 이상) 종목 진입 원천 차단, 손절 후 60분 쿨다운 강제(Revenge Trading 방지), Track A 일일 손절 4회 누적 시 당일 Track A 비활성화.

6. Execution Instruction (운용 지침)
KIS 일봉 API는 1회 최대 100건 한계가 있으므로, 장기 데이터(Track E 250일+/F 200일+)는 get_daily_chart의 자동 페이지네이션(최대 5페이지)으로 수집한다.
Track E 스캔 시 Pandas rolling max 연산을 통해 200일 최고가를 정확히 산출한다.
Track E 후보는 당일 -3% 이상 급락 중이어도 사전 필터 SKIP에서 면제된다 (급락 자체가 전제 조건).
Track F는 매 거래일 14:50~15:05에 1회씩, 방어선(200일선 기반) 위에서 구조 유지 시 다음 차수를 종가 매집한다 (총 5분할).
모든 트랙은 '영웅문 정량 필터'를 먼저 통과한 종목에 대해서만 AI 라우팅을 진행한다 (Hard Filter 정책).
Track C는 14:45분에 1차 검색을 수행하고, 미충족 시 15:00에 최종 검색하여 종목을 발굴한다.
Track A 피벗 스나이퍼(Phase 1.5)는 속도 우선 경로로 Gemini 펀더멘털 검증을 생략하되, 로컬 ML 승률 게이트는 통과해야 한다.
Track C/D/F 관심종목(Watchlist)은 ML 모델 보유 시 승률 70% 기준으로 격발하고, 모델 학습 전에는 룰 기반(종배: 14:45 도달 / 스윙·매집: AI 진입가 +1% 이내 도달)으로 격발한다.
치명적 에러(토큰 403 등) 발생 시 보유 포지션을 청산하지 않고 유지한 채 재시작한다 (인프라 에러로 인한 투매 금지 원칙).
